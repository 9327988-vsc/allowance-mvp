# HexStack — 개발 워크플로우

기존 서비스 개선과 신규 개발을 모두 다루는 단계별 워크플로우. 롤백 안전성·외과적 변경·검증 게이트를 내장한다.

---

## 모드 선언 (필수)
프로젝트 시작 시 선언한다.
```
MODE=new       # 신규 프로젝트
MODE=improve   # 기존 서비스 개선
```

## 태스크 규모 선언
```
TASK_SCALE=small    # 버그 수정, 단순 기능 추가
TASK_SCALE=medium   # 기능 개선, 모듈 단위 변경
TASK_SCALE=large    # 대규모 리팩토링, 신규 서비스
```

| 규모 | git commit 시점 | 승인 게이트 |
|------|----------------|------------|
| small | 코딩 전 1회 | 생략 가능 |
| medium | 설계·코딩·통합 전 | 설계·통합 시 필요 |
| large | 매 단계 전 | 매 단계 필요 |

---

## 단계 순서

### MODE=new
```
기획 → 설계 → 디자인 → 코딩
→ 검수 → 테스트 → 통합 → 보안 → 배포 → 회고
```

### MODE=improve
```
기획 → 현황 분석 → 변경 범위 확정
→ 코딩 → 검수 → 테스트 → 통합
→ 보안 → 배포 → 회고
```

---

## 절대 금지 명령
환경에 따라 조정하되, 기본 금지 목록은 다음과 같다.
```
⛔ 장시간 점유 서버 실행 명령 (npm run dev, npm run build, node index.js 등)
⛔ kill, pkill, lsof (프로세스 강제 종료)
⛔ git push --force (main/master)
⛔ DROP TABLE (보안 승인 없이)
⛔ git checkout -- . (전체 롤백 금지 — 파일별 선택 롤백만 허용)
⛔ --no-verify (hook 우회)
⛔ git rebase -i, git add -i (interactive 명령)
```

## 단계 시작 전 필수 규칙

### 규칙 1 — git commit 의무화
모든 코딩 단계 시작 전 커밋한다.
```bash
git add -A
git commit -m "before-[단계명]: [작업 설명]"
```
미커밋 상태이면 작업을 중단하고 커밋을 요청한다.

### 규칙 2 — 변경 파일 목록 사전 출력
코딩 단계 시작 전 아래 형식으로 출력하고 승인을 받는다.
```
## 변경 예정 파일
- 파일1 — 변경 내용 (영향 범위: 낮음/중간/높음)
- 파일2 — 변경 내용 (영향 범위: 낮음/중간/높음)

변경 파일 수: N개 | 예상 변경 줄 수: 약 N줄
롤백 방법: git checkout -- 파일1 파일2

진행하시겠습니까? (Y 입력 시 시작)
```
사용자가 Y를 입력해야만 코드 수정을 시작한다. N이면 중단한다.

### 규칙 3 — 수동/자동 편집 모드 명시
```
EDIT_MODE=manual   # 최소 수정 (기본값)
EDIT_MODE=auto     # 자동 편집 도구 사용 — 사용 시 반드시 git commit 후 진행
```

## 현황 분석 (improve 모드)
변경 전 아래를 먼저 수행한다.
```
## 현황 분석
- 변경 대상 파일: [목록]
- 영향받는 파일: [목록]
- 현재 정상 동작 기능: [목록]
- 변경 시 깨질 수 있는 기능: [목록]
- 위험도: 높음/중간/낮음
```
위험도가 높으면 TASK_SCALE 상향을 검토한다.

---

## 테스트 환경 명시
프로젝트 시작 시 선언한다.
```
## 테스트 환경
- 서버 실행: 가능 / 불가능
- DB 직접 접근: 가능 / 불가능
- 브라우저 자동화: 가능 / 불가능
- 테스트 방법: 실행 테스트 / 코드 리뷰 / 정적 분석
```
서버 실행이 불가한 환경에서는 코드 리뷰 + 정적 분석만 수행한다고 명시한다.

---

## 승인 게이트
```
[GATE] Phase N → Phase N+1

## 변경 예정 파일
- 파일명 — 변경 내용 (영향 범위)

## 현재 git 상태
커밋 완료 / 미커밋 N개 파일 (커밋 후 진행 필요)

## 롤백 방법
git checkout -- [파일목록]

승인: Y 입력 시 진행
```

---

## 단계 완료 기준 (PHASE_SUMMARY)
```
## Phase N 완료 — [단계명]
- 완료 일시:
- 커밋 해시: (before/after)
- 변경된 파일 목록:
- 롤백 방법:
- 다음 단계 진입 조건:
- 미해결 이슈:
```

## 세션 종료 시 산출물
1. 변경 파일 목록 + 커밋 해시
2. 다음 세션 브리핑 (3줄)
3. 미완료 태스크 목록
4. 미커밋 파일 목록 (있으면 커밋 권고)

---

## 검수 점검 항목
코드 변경 후 자체 검수 시 점검한다.
- 경쟁 조건(race condition) 방어
- 에러·실패 복구 경로
- 프레임워크 관용·규칙 준수
- 기존 컨벤션 위반 여부
- 누락된 엣지 케이스
- 보안 체크 (CODING_RULES 규칙 3)
