# POLICIES — 항구적 결정 + Phase 로드맵

## 정책 우선 원칙
**"추후 과제 ≠ 정책 없음."** 구현 시점이 미래여도 정책은 현 시점에 확정하고 문서화한다.

## 정책이 반드시 포함하는 3가지
1. **(a) 최종 목표 정책** (완전판)
2. **(b) 구현 로드맵** (Phase 1·2·3)
3. **(c) 현재 구현 상태** (어느 Phase까지 구현됐는지)

## 코드 참조 주석 (필수)
정책과 코드를 연결한다.
```js
// 정책 #NNN — 제목 (Phase X 구현 중)
// TODO[정책 #NNN, Phase Y]: (다음 단계 작업 설명)
// 참조: /POLICIES.md
```

## Phase 상승 트리거
"언제 다음 Phase를 착수할지"의 판단 기준을 정책에 명시한다.

## 정책 #001 — 인증 방식
- 결정일: 2026-05-21

### (a) 최종 목표 정책
- 아이디: 영문 시작, 영문+숫자, 3~20자, 대소문자 무시 중복 검사
- 비밀번호: 영문+숫자 필수, 특수문자·대문자 허용, 8자 이상
- 비밀번호 초기화: 가입 시 설정한 보안 질문 답변으로 초기화
- 해시: PBKDF2-SHA256, 100,000회 반복, per-user salt
- Brute-force 방어: 5회 실패 → 30초 잠금

### (b) 구현 로드맵
- Phase 1: PIN → 아이디+비밀번호 전환 (기본 인증)
- Phase 2: 레거시 PIN 코드 완전 제거
- Phase 3: (향후) 이메일 인증, OAuth 등 추가 인증 수단

### (c) 현재 구현 상태
